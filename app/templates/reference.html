Цель и задачи проекта

  • Разработка веб-приложения для каталога и продажи серверного оборудования.
  • Автоматизация оформления заказов и управления складскими остатками.
  • Реализация ролей пользователей: Администратор, Модератор, Пользователь.
  • Генерация PDF-отчётов по пользователям и по серверам.
  • Обеспечение безопасности и удобного интерфейса.

Стек технологий

  • Язык программирования: Python 3.10+
  • Веб-фреймворк: Flask
  • ORM: SQLAlchemy + Flask-Migrate (Alembic)
  • СУБД: PostgreSQL 14
  • Контейнеризация: Docker & Docker Compose
  • Асинхронность: Flask-SocketIO (для чата и уведомлений в реальном времени)
  • Аутентификация: Flask-Login + JWT (для API)
  • Формы и валидация: Flask-WTF + WTForms
  • Генерация PDF: ReportLab
  • Шаблонизация: Jinja2, Bootstrap 5 (CSS/JS), AJAX (Fetch API)
  • Тестирование: pytest, Flask-Testing, SQLite (для CI)
  • Статика и загрузки: Flask’s static, WTForms FileField, uploads/

Архитектура приложения

  1. Blueprints:
    • main — публичная часть (каталог серверов).
    • user — функционал клиента (корзина, заказы, профиль).
    • moderator — управление заказами, чат.
    • admin — CRUD по серверам, пользователям, отчётам, статистика.
  2. Модель данных: User, Server, Order, OrderItem, ChatMessage, ReportUser/ReportServer, LoginStat.
  3. API: JSON-эндпоинты для деталей сервера, корзины и заказов.
  4. Роуты и роли: разграничение доступа через @login_required и @admin_required.

Основные функции

  • Добавление и редактирование серверов (с выбором URL или загрузкой файла изображения).
  • Управление складскими остатками и флагом «в наличии».
  • Оформление заказов напрямую или через корзину с учётом доступного количества.
  • Реализация ролей с разными правами (USER, MODERATOR, ADMIN).
  • Онлайн-чат внутри заказов (SocketIO).
  • Генерация и скачивание PDF-отчётов.
  • Валидация JSON для характеристик серверов (CodeMirror + клиентская/серверная проверка).
  • Система уведомлений (флеш-сообщения и тосты).
